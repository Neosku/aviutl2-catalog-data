# パッケージ登録方法

## 登録画面の開き方
本ソフト → 上部メニューの **「フィードバック・パッケージ登録」** → **「パッケージ登録」**

---

## 各項目の説明

| 項目名 | 説明 |
|--------|------|
| **ID** | パッケージの識別子です。英数字が使用できます。<br>形式は `作者名.パッケージ名` としてください。<br>例：`Kenkun.AviUtlExEdit2` |
| **パッケージ名** | パッケージの名称を入力します。日本語も使用可能です。 |
| **作者名** | パッケージの作者名を入力します。日本語も使用可能です。 |
| **オリジナル作者名（任意）** | 元となるパッケージの作者名を入力します。日本語も使用可能です。 |
| **種類** | パッケージの種類を入力します。<br>例：入力プラグイン / 出力プラグイン / スクリプト / 言語ファイル など<br>自由入力なので、上記に該当するものがあればそれを入力してください。 |
| **概要** | パッケージの概要を55文字以内で入力してください。 |
| **詳細** | パッケージの詳細な説明を **Markdown形式** で入力してください。 |
| **パッケージのサイト** | パッケージに関するページのURLを入力します。<br>X（旧Twitter）やニコニコ動画のリンクでも構いません。 |
| **ライセンス** | パッケージのライセンスを入力してください。 |
| **タグ** | タグを入力します。複数指定可能で、自由に設定できます。 |
| **依存パッケージ** | 依存関係のあるパッケージIDを入力してください。<br>※現在は未対応のため、入力しても動作には影響しません。 |
| **サムネイル画像** | ホーム画面で表示される画像です。<br>10MB以下の PNG または JPG を1枚アップロードしてください。 |
| **説明画像** | パッケージ詳細ページの上部に表示される画像です。複数枚選択可能です。 |

## ダウンロード元の指定方法

### 1. 直接URLの場合
直接ダウンロードできるファイルのURLを入力してください。

### 2. GitHub Releaseの場合
- **GitHubアカウント名**
- **レポジトリ名**
- **ダウンロードファイル名**（正規表現対応）

ダウンロードするリリースは「最新のタグが付いたリリース」です。

### 3. Google Driveの場合
URLではなく **ファイルID** を入力してください。  
Google Drive上でファイルを開いたとき、URL末尾にある文字列がファイルIDです。

## インストールステップ

インストール時の処理には以下の4種類があります。

1. **ダウンロード**
2. **ZIP展開**
3. **EXE実行**
4. **コピー**

### ダウンロード,ZIP展開
ダウンロードと ZIP 展開は、インストール実行時に自動作成される一時フォルダ`{tmp}`で行われます。

### EXE実行
- 実行パスには下記の **変数** を使用してください。
- 実行時に渡される引数も指定できます。

### コピー
- コピー元、コピー先を指定してください。
- パスの指定には **変数** を使用してください。

## バージョン管理について

本ソフトでは、バージョン管理に **ハッシュ値** を使用します。  
主要なファイル（例：プラグインの場合 `.aui2` や `.auo2`）のみを選択し、ハッシュ値を計算してください。

### 保存先パスの指定
AviUtl内での保存先を指定します。  
例：  
```text
{pluginsDir}/al2_jd.aui2
```

### 使用可能な変数
以下の変数が使用可能です。

- `{tmp}`: パッケージごと、バージョンごと自動で作成される一時フォルダです。　ダウンロード先やZIP展開先として使用します。
- `{pluginsDir}`: プラグイン配置先
- `{scriptsDir}`: スクリプト配置先
- `{dataDir}`: aviutlのデータ保存先（Language, Plugin, Scriptの親フォルダ）
