# ニコニ・コモンズ作品ID一覧出力プラグイン

AviUtl2のプロジェクト内で使用されている親作品登録可能な作品ID（nc/im/sm/ss で始まるID）を自動取得し、テキストファイルに出力する汎用プラグイン（.aux2）です。

詳しい説明は解説動画を作成しているのでそちらをご参照ください → https://www.nicovideo.jp/watch/sm46156601

## 主な機能

- アクティブシーン内の動画・画像・音声・PSDファイルオブジェクトから作品IDを自動取得
- 重複を除去し、昇順ソートして一覧化
- クリップボードへのコピー、テキストファイルへの保存に対応
- 日本語・英語の対応

## 動作環境

- Windows 10/11 (64bit)
- AviUtl2

## インストール（Aviutl2カタログからの場合）

インストールボタンを押すと登録完了です

## インストール（自分で入れる場合）

1. [Google drive](https://drive.google.com/drive/folders/17BQSHUlCVBaRFADDYpjd_eH8UJ_FZ69x?usp=sharing) から最新の `commons_material_extractor.aux2` をダウンロード
2. ダウンロードしたファイルを `aviutl2\Plugin\` に配置
3. AviUtl2 を再起動

正しくインストールされていれば「表示」>「親作品登録可能な作品IDを取得」が表示されます。

## 使い方

1. AviUtl2 で作品IDを含むプロジェクトを開く
2. メニューバーから「設定」→「親作品登録可能な作品IDを取得」を選択し、プラグインのウィンドウを開く
3. 「取得」ボタンをクリックすると、作品ID一覧がテキストボックスに表示される
4. 「コピー」でクリップボードにコピー、「ファイルに保存」でテキストファイルに出力

### 出力例

```text
im300
nc100
nc200
sm500
ss600
td700
```


## オプション

| オプション | デフォルト | 説明 |
| --- | --- | --- |
| フォルダー名も取得対象とする | OFF | ONにすると、ファイル名だけでなくフォルダー名に含まれるIDも取得する |
| プロジェクト全体を取得対象とする | OFF | ONにすると、アクティブシーンだけでなく全シーンからIDを取得する |

- オプションの状態はプラグインDLLと同じディレクトリに `commons_material_extractor_settings.json` として自動保存されます
- チェックボックスを変更すると即座に保存され、次回起動時に復元されます
- 両方のオプションは独立して動作します。組み合わせによる動作は以下の通りです:

| プロジェクト全体 | フォルダー名 | 動作 |
| --- | --- | --- |
| OFF | OFF | アクティブシーンのファイル名のみからIDを取得 |
| OFF | ON | アクティブシーンのファイルパス全体（フォルダー名含む）からIDを取得 |
| ON | OFF | プロジェクト全体のファイル名のみからIDを取得 |
| ON | ON | プロジェクト全体のファイルパス全体（フォルダー名含む）からIDを取得 |

## 対応するIDプレフィックス

| プレフィックス | 説明 |
| --- | --- |
| `nc` | ニコニ・コモンズ素材 |
| `im` | ニコニコ静画 |
| `sm` | ニコニコ動画 |
| `ss` | ニコニコショート |

正規表現 `(nc|im|sm|ss|td)\d+` にマッチするパターンを作品IDとして認識します。

## 取得パターン

### 取得可能なパターン

| パターン | 例 | 取得されるID |
| --- | --- | --- |
| ファイル名がIDで始まる | `nc12345.wav` | `nc12345` |
| ファイル名の途中にIDがある | `立ち絵_im10585578.psd` | `im10585578` |
| サフィックス付きファイル名 | `nc100_2.mp3` | `nc100` |
| 絶対パスのファイル名部分 | `C:\素材\sm999.mp4` | `sm999` |
| 拡張子なしのファイル名 | `nc42` | `nc42` |
| 各種拡張子 | `.wav`, `.mp3`, `.png`, `.bmp`, `.mp4`, `.psd` など | 拡張子に依存しない |

### 「フォルダー名も取得対象とする」ONの場合に追加で取得可能

| パターン | 例 | 取得されるID |
| --- | --- | --- |
| フォルダー名にIDがある | `E:\im10585578\立ち絵.psd` | `im10585578` |
| フォルダー名とファイル名の両方にIDがある | `C:\nc100\im200.png` | `nc100`, `im200` |

### 「プロジェクト全体を取得対象とする」ONの場合

プロジェクトファイルを直接読み込み、全シーン（Root・サブシーン含む）に散らばったIDを一括取得します。OFFの場合はアクティブシーンのみが対象です。

このオプションがONの場合、ファイル保存時のデフォルトファイル名にはプロジェクトファイル名が使用されます（OFFの場合はアクティブシーン名）。

### 取得されないパターン

| パターン | 例 | 理由 |
| --- | --- | --- |
| 大文字のプレフィックス | `NC12345.wav` | 小文字の `nc`/`im`/`sm`/`ss` のみ対象 |
| プレフィックスの後に数字がない | `ncfile.txt` | `(nc\|im\|sm\|ss)\d+` にマッチしない |
| IDを含まない通常ファイル | `background.png` | IDパターンが含まれない |
| フォルダー名のみにIDがある（オプションOFF時） | `D:\im10585578\立ち絵.psd` | デフォルトではファイル名のみ対象 |

## トラブルシューティング

| 状況 | 原因と対処 |
| --- | --- |
| 「プロジェクトファイルの読み込みに失敗しました。」と表示される | プロジェクトファイルが見つからないか、アクセスできない状態です。プロジェクトを再度開いてからもう一度実行してください |
| 「プロジェクトファイルが開かれていません」と表示される | プロジェクト全体取得がONの状態で、プロジェクトが未保存または未読み込みです。先にプロジェクトを保存してから実行してください |
| 「クリップボードへのコピーに失敗しました」と表示される | 他のアプリケーションがクリップボードをロックしている可能性があります。しばらく待ってから再度コピーしてください |
| 設定（チェックボックス）が保存されない | プラグインDLLのあるフォルダーに書き込み権限がない可能性があります。Aviutl2のPlugin配下 の権限を確認してください |


## 対象エフェクト

以下のエフェクトからファイルパスを取得し、作品IDを抽出します:

| エフェクト名 | プロパティキー |
| --- | --- |
| 動画ファイル / VideoFile | ファイル / File |
| 画像ファイル / ImageFile | ファイル / File |
| 音声ファイル / AudioFile | ファイル / File |
| PSDファイル@PSDToolKit / PSDFile@PSDToolKit | PSDファイル / PSDFile |
